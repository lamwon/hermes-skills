# AI Philosophy Oracle Website Builder

Build an interactive "philosophy oracle" website where users:
1. Select a life problem (categorized or free input)
2. Select a philosopher/school (can be deliberately mismatched)
3. Get AI-generated Feynman-style analysis streamed via SSE
4. Conclude with Buddhism/Daoism/Confucianism quote + generated image
5. Share to WeChat Moments or save as image

**Key differentiator**: NO content database — everything dynamically generated by LLM.

## Philosophy Content Framework

### Philosopher system: Russell's "A History of Western Philosophy"

Three-volume structure: all major figures must be covered.

**Book I: Ancient Philosophy**
- Pre-Socratics: Thales, Anaximander, Anaximenes, Pythagoras, Heraclitus, Parmenides, Empedocles, Anaxagoras, Democritus (Atomists), Protagoras (Sophists)
- The Three: Socrates, Plato, Aristotle
- Post-Aristotelian: Cynics (Diogenes), Stoics (Zeno, Marcus Aurelius), Epicureans, Sceptics (Pyrrho)
- Neoplatonism: Plotinus

**Book II: Catholic Philosophy**
- Patristic: Augustine
- Scholastic: Thomas Aquinas, Duns Scotus, William of Ockham

**Book III: Modern Philosophy**
- Renaissance: Machiavelli, Erasmus, More, Bacon
- Rationalists: Descartes, Spinoza, Leibniz
- Empiricists: Locke, Berkeley, Hume
- Enlightenment: Rousseau, Kant
- German Idealism: Hegel, Schopenhauer, Nietzsche
- Utilitarianism: Bentham, Mill
- Pragmatism: James, Dewey
- Analytic Philosophy: Frege, Russell, Wittgenstein
- Others: Byron, Marx, Bergson

### Philosopher JSON config (minimal: used for AI prompt injection)

```typescript
interface Philosopher {
  id: string;           // e.g. "kant"
  name: string;         // "康德"
  nameEn: string;       // "Immanuel Kant"
  era: "ancient" | "medieval" | "modern";
  volume: 1 | 2 | 3;    // Russell volume
  school: string;       // "义务论" / "理性主义" / etc.
  summary: string;      // 50-word core idea (for prompt context)
  keyWork: string;      // <纯粹理性批判>
  famousQuote: string;  // "要这样行动..."
  oppositeIds: string[]; // opposing philosopher IDs for "换个角度" section
}
```

## Core Interaction Flow (Four-Step State Machine)

```
Step 1: 选问题
  - 预设分类: 职场困惑 / 情感关系 / 消费决策 / 道德困境 / 人生意义 / 自定义
  - 自由输入文本框 + 分类点选
  - State: { step: 1, question: string, category?: string }

Step 2: 选哲学家
  - 按罗素三卷本分类浏览
  - 搜索/筛选
  - "随机试试"按钮
  - 可故意选不相关的 (mismatch mode)
  - State: { step: 2, philosopherIds: string[], isMismatch: boolean }

Step 3: 看分析 (SSE流式输出)
  - POST /api/analyze { question, philosophers, isMismatch }
  - SSE chunks: { type: "feynman" | "analogy" | "deep" | "advice" | "alternative" | "done" }
  - 打字机效果逐字渲染
  - 心跳机制 (每10s): { type: "heartbeat" }
  - State: { step: 3, analysis: StreamState, complete: boolean }

Step 4: 佛道儒结语
  - 佛 / 道 / 儒 三选一
  - POST /api/wisdom { tradition, analysisContext }
  - 返回: { quote, source, explanation, imageDescription }
  - 生成分享卡片 (前端Canvas API)
  - Share: navigator.share() or download as image
```

### Prompt Engineering: Two Modes

**Match mode** (default):
```
请用{哲学家}的思想分析问题: {问题}
按以下费曼格式输出:
1. 一句话概括 (小学生能懂)
2. 生活类比
3. 深度分析 (200字)
4. 生活建议
5. 换个角度 (用{opposite}的思想)
```

**Mismatch mode** ("驴唇不对马嘴"):
```
{哲学家}的思想通常与{问题}无关。
请指出{哲学家}会如何看待这个问题--
他/她可能觉得这不属于其关注领域,
但尝试用其方法论给出一个出乎意料的答案。
保持思辨趣味, 格式同费曼四步法。
```

## Technical Architecture

### Stack
- **Frontend**: Next.js 14+ (App Router) + Tailwind CSS v4 + shadcn/ui
- **AI**: DeepSeek V4 Flash via cc-switch proxy (127.0.0.1:15721)
- **Streaming**: SSE (Server-Sent Events) from Next.js API Routes
- **Sharing**: Canvas API (frontend-only, avoid serverless canvas rendering)
- **State**: useReducer for 4-step FSM + URL search params for shareable links
- **Deploy**: Vercel (free tier: no database, no persistent storage)

### API Endpoints
| Endpoint | Method | Purpose | Streaming |
|----------|--------|---------|-----------|
| /api/analyze | POST | {question, philosopherIds, isMismatch} -> Feynman analysis | SSE |
| /api/wisdom | POST | {tradition, context} -> quote + image description | No |
| /api/philosophers | GET | Return philosophers.json | No |
| /api/schools | GET | Return school/era taxonomy | No |

## Common Pitfalls

1. **"高星星" is slang for high stars**: User may say "高星星的skill" meaning high-starred skills, not a person. Search by star count.
2. **GitHub API blocked**: Use ddgs text search or skills.sh instead of api.github.com for discovery in China.
3. **SSE heartbeat required**: DeepSeek's generation can pause >15s between chunks. Send heartbeat every 10s to prevent browser timeout.
4. **Canvas on serverless = bad**: Share cards should be generated client-side (Canvas API in browser), not via Serverless API route which can't load fonts.
5. **Two prompt templates needed**: Match mode vs mismatch mode are fundamentally different prompt strategies.
6. **python-docx on Windows**: Use Hermes embedded Python (python_embedded/python.exe), not system python3 from Windows Apps (exit code 49 on imports).
7. **Chinese path names**: Project directory can contain Chinese characters; subagent tools may fail on first attempt, use search_files to resolve real path.
